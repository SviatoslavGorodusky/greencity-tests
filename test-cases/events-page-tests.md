# Тест-кейси для сторінки подій

### 1. Фільтрація подій за типом
| Крок | Дія | Очікуваний результат |
| :--- | :--- | :--- |
| 1 | Відкрити сторінку GreenCity Events | Сторінка завантажена, відображаються доступні події |
| 2 | Натиснути на випадаючий список "Type" | З'являється список доступних типів подій |
| 3 | Обрати чек-бокс "Economic" | Фільтр активується, список подій оновлюється |
| 4 | Перевірити наявність тегу "Economic" | Під заголовком з'являється зелений тег "Economic" |
### 2. Позитивна перевірка пошуку подій за назвою
| Крок | Дія                                                       | Очікуваний результат |
| :--- |:----------------------------------------------------------| :--- |
| 1 | Натиснути на іконку пошуку (лупа)                         | З'являється поле для введення текстового запиту |
| 2 | Ввести валідну назву існуючої події (напр. "Some Events") | Список подій миттєво фільтрується |
| 3 | Перевірити назву першої картки у результатах              | Назва знайденої події збігається з пошуковим запитом |

### 3. Перевірка перемикання вигляду (Grid/List)
| Крок | Дія | Очікуваний результат |
| :--- | :--- | :--- |
| 1 | Знайти перемикач вигляду (справа над списком подій) | Відображаються дві іконки: сітка та список |
| 2 | Натиснути на іконку "List View" | Картки подій перешиковуються у вертикальний список |
| 3 | Натиснути на іконку "Grid View" | Картки подій повертаються до стандартного вигляду сітки |

### 4. Негативна перевірка пошуку
| Крок | Дія | Очікуваний результат |
| :--- | :--- | :--- |
| 1 | Активувати поле пошуку | Поле готове до введення тексту |
| 2 | Ввести набір випадкових символів (напр. "@@@###") | Система обробляє запит і не знаходить збігів |
| 3 | Перевірити центральну частину сторінки | Відображається повідомлення "No events found" або аналогічне |
